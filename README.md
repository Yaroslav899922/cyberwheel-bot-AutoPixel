# 🏎️ CyberWheel AI Engine (v2.7 Anti-Duplicate Edition)

**CyberWheel** — це повністю автономна інтелектуальна система кураторства контенту. Бот самостійно моніторить провідні світові та українські медіа, аналізує новини за допомогою штучного інтелекту та створює експертні дайджести для Telegram-каналу. 

Головна дійова особа системи — **Агент Софія**, віртуальний ШІ-аналітик, який адаптує контент, робить професійні висновки та взаємодіє з аудиторією.

---

## 🎯 Мета проекту (The Vision)
Створити безперервний конвеєр обробки даних, який перетворює "сирий" потік новин на структурований, візуально досконалий та корисний контент для бізнесу (автосалону) без участі людини.

## 🛠️ Технологічний стек
* **Мова:** Python 3.11+
* **ШІ:** Google Gemini API (модель Flash-Lite, fallback на 3 моделі)
* **База даних:** Upstash Redis (інтеграція через нативний `urllib` POST)
* **Фреймворки:** Flask (багатопотоковий веб-сервер)
* **Інструменти:** Feedparser, Newspaper3k, Schedule, Регулярні вирази (re)
* **Хостинг:** Render.com + GitHub (auto-deploy)

### v2.7 (Anti-Duplicate Edition) — *Поточна версія* — 13.04.2026

#### 🔁 Семантичний фільтр дублів статей
Стара система порівнювала заголовки як один злитий рядок — не ловила різні статті про одну тему з різних сайтів (наприклад, три матеріали про рекорд Ford GT з motor1, caranddriver та autocar).
* **`TOPIC_STOP_WORDS`** — список ~60 загальних слів (артиклі, прийменники, типові авто-слова) які не несуть унікальності теми
* **`get_topic_words()`** — витягує значущі слова з заголовку у вигляді множини
* **`is_title_duplicate()`** — блокує статтю якщо ≥3 ключових слів вже є в Redis (`tw:{слово}`)
* **`save_title_fingerprint()`** — зберігає кожне значуще слово окремо в Redis на 7 днів після публікації

#### 🛡️ Захист від дублів при деплої
Виявлено: при кожному деплої Render запускає два екземпляри бота одночасно на 1-2 хвилини. Обидва проходили початкову перевірку і публікували одну й ту саму статтю.
* **`is_url_already_published()`** — фінальна перевірка Redis командою `SISMEMBER` прямо перед відправкою в Telegram. Перший екземпляр публікує і записує URL, другий — тихо зупиняється.

#### 🏆 Тижневий дайджест: ТОП-7 з розумним відбором
Стара система кидала всі 200+ заголовків у Gemini — він надавав перевагу свіжішим через їх позицію в кінці списку, а не реальну важливість.

Нова система — три етапи перед Gemini:
* **Дедублікація** — прибирає статті на одну тему (≥3 спільних слова між заголовками)
* **Система балів** — автоматичний пріоритет за категоріями:
  * Škoda / моделі → `+10` (завжди в дайджест)
  * Ринок/продажі/статистика **України** → `+10` (рівень Škoda, обов'язково)
  * Електромобілі / нові технології → `+5`
  * Рекорди / прориви → `+4`
  * Ринок/продажі (не Україна) → `+3`
  * Ціни/акції **України** → `+2`
  * Ціни/акції **не України** → виключаються повністю (`-1`)
* **Топ-30** — тільки найвищі за балами йдуть до Gemini, який робить фінальний вибір 7 новин з різноманітністю категорій
* `MAX_STORED` збільшено з 100 до 200 записів

### v2.6 (Weekly Edition) — 29.03.2026
* **Тижневий ТОП-5 дайджест:** новий файл `weekly_digest.py`. При кожній публікації заголовок зберігається в Redis LIST `weekly_headlines`. Щонеділі о 21:45 Gemini обирає топ-5 найважливіших новин тижня з пріоритетом для Škoda, формує дайджест з клікабельними заголовками та вердиктом Агента Софії.
* **Клікабельні заголовки у дайджесті:** формат `<b><a href="url">назва</a></b>` — жирний, синій, клікабельний одночасно.
* **Автоматичний діапазон дат:** "23–29 березня" або "28 березня – 3 квітня" при переході місяця.
* **Fallback моделей у weekly_digest.py:** три моделі Gemini по черзі — як в `brain.py`.
* **Маркер тижня:** `last_weekly_digest.txt` зберігає номер тижня (`%Y-W%W`) для захисту від повторної відправки.
* **Тестовий маршрут:** `/test-weekly` у Flask для перевірки дайджесту без Shell (безкоштовний Render).
